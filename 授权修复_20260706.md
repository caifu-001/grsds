## 授权修复 2026-07-06

### 根因
`batch_grant_resources` RPC 接受 `p_resource_ids INTEGER[]`，前端传的是 JS 数组 `[1]`，经 `admin-proxy` → JSON → PostgREST 链路，数组丢了，`FOREACH` 遍历空数组导致 `resource_id = null`，触发 NOT NULL 约束。

此外 `auth.uid()` 在 service key（admin-proxy）调用下返回空，导致 `granted_by = null`。

### 修复
1. **RPC 重写** (`sql/fix_batch_grant.sql`)：参数改为 `p_resource_ids TEXT`（逗号分隔 `"1,3,5"`），用 `string_to_array` 拆分。`granted_by` 增加可选参数 `p_granted_by`，默认 `COALESCE(p_granted_by, p_user_id)`。
2. **前端** (`saveMemberGrants`)：`ids.join(',')` 拼接字符串，传 `p_granted_by: currentUser.id`。
3. **时序修复**：`loadMemberGrants()` 移到 `loadClients()` 之前 + await。
4. **错误处理增强**：`saveMemberGrants` + `loadMemberGrants` 添加 console.log 诊断日志。
